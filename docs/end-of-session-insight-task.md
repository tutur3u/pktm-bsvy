# End-Of-Session Insight Task

## Goal
Capture one durable insight at the end of each session so future changes start with better context.

## When To Run It
- After implementation and verification are complete.
- Before sending the final user summary.

## What To Record
- One concise insight from the session.
- Prefer repo-specific facts, debugging shortcuts, or design implementation lessons.
- Avoid generic advice that would be true in any project.

## Output Location
- Append the new note to `docs/session-insights.md`.

## Format
- `YYYY-MM-DD: insight`

## Good Examples
- `2026-04-20: Desktop hero composition depends on overlaying bs-vy.png on top of hero.png; using hero.png alone drops the doctor entirely.`
- `2026-04-20: The press section should render supplied article screenshots directly without extra card chrome or the mock drifts immediately.`

## Update Rule
- If the routine changes, update both this file and `AGENTS.md` together.
